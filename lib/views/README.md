# Views Directory

This directory contains all screen-level UI components and pages.

## Purpose:
- Implement complete screen layouts
- Handle user interactions at the screen level
- Coordinate between providers and widgets
- Manage navigation between screens
- Implement screen-specific business logic

## Examples:
- Home screen
- News detail screen
- Settings screen
- Profile screen

## Best Practices:
- Keep views focused on layout and navigation
- Use providers for state management
- Break down complex UIs into smaller widgets
- Implement proper responsive design
- Handle screen-specific error states
- Use const constructors where possible
