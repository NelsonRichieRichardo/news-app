# Models Directory

This directory contains all data models and entities used in the application.

## Purpose:
- Define data structures for API responses
- Create Dart classes that represent real-world entities
- Include serialization/deserialization methods (fromJson/toJson)
- Handle data validation and transformation

## Examples:
- User model
- Article/News model  
- Weather data model
- API response models

## Best Practices:
- Keep models focused on data representation only
- Include proper null safety
- Add documentation comments for complex fields
- Use factory constructors for JSON parsing
